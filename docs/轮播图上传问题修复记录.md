# 轮播图上传功能问题修复记录

## 问题日期
2026-04-02

## 问题描述
轮播图上传功能存在两个问题：
1. **上传失败**：前端无法成功上传图片到后端
2. **图片无法查看**：即使上传成功，前端也无法访问到已上传的图片（403 Forbidden）

---

## 问题根因分析

### 问题 1：前端 API 路径错误

**现象**：前端请求 `http://localhost:8081/api/api/admin/banners/upload`（路径重复）

**原因**：
- `api.ts` 中 `API_ADMIN_URL = 'http://localhost:8081/api/admin'`
- `Banners.tsx` 中直接使用 `fetch(`${API_BASE_URL}/api/admin/banners/upload`)`
- 导致路径拼接重复：`/api` + `/api/admin/...` = `/api/api/admin/...`

**修复**：
```typescript
// Banners.tsx 第 150 行
// 修复前：
const response = await fetch(`${API_BASE_URL}/api/admin/banners/upload`, {

// 修复后：
const response = await fetch(`${API_BASE_URL}/admin/banners/upload`, {
```

---

### 问题 2：静态资源访问 403 Forbidden

**现象**：访问 `http://localhost:8081/uploads/banners/xxx.png` 返回 403

**原因**：Spring Security 请求匹配规则**顺序问题**

```java
// 修复前的顺序（错误）
.requestMatchers("/api/admin/**").permitAll()
.requestMatchers("/api/**").authenticated()      // ⚠️ 这条在前
.requestMatchers("/uploads/**").permitAll()      // ⚠️ 这条在后，永远不会匹配到
```

当访问 `/api/uploads/banners/xxx.png` 时：
1. Spring Security 从上到下匹配规则
2. `/api/uploads/...` 先匹配到 `/api/**` 规则
3. 触发 `.authenticated()` 要求认证
4. 静态资源请求没有 Token，返回 **403 Forbidden**

**修复**：
```java
// SecurityConfig.java - 调整顺序
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
        .cors(cors -> cors.configurationSource(corsConfigurationSource()))
        .csrf(AbstractHttpConfigurer::disable)
        .sessionManagement(session -> session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
        .authorizeHttpRequests(auth -> auth
            // ✅ 静态资源必须在 /api/** 之前
            .requestMatchers("/uploads/**", "/api/uploads/**").permitAll()
            .requestMatchers("/api/admin/auth/**").permitAll()
            .requestMatchers("/api/admin/**").permitAll()
            // ... 其他配置
            .requestMatchers("/api/**").authenticated()
            .anyRequest().permitAll()
        )
        .addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);
    return http.build();
}
```

### 问题 3：上传路径配置错误

**现象**：文件保存到错误目录 `fitness-java/fitness-java/uploads/`

**原因**：
```yaml
# application.yaml
upload:
  path: ${user.dir}/fitness-java/uploads  # ⚠️ user.dir 在 Maven 运行时指向父目录
```

当通过 `mvn spring-boot:run` 启动时：
- `${user.dir}` = `/home/zfxt/formyself/fitness-admin-all/fitness-java`
- 实际路径 = `/home/zfxt/formyself/fitness-admin-all/fitness-java/fitness-java/uploads/`
- 但 `WebConfig.java` 配置的资源路径指向单层目录，导致找不到文件

**修复**：
```yaml
# application.yaml - 使用绝对路径
upload:
  path: /home/zfxt/formyself/fitness-admin-all/fitness-java/uploads
```

## 关键知识点

### Spring Security 规则匹配顺序
- **从上到下**依次匹配
- **第一个匹配成功**的规则生效
- 静态资源、登录接口等**免认证**路径必须放在**前面**
- 需要认证的路径放在**后面**

### 文件上传路径配置
- 避免使用 `${user.dir}` 等相对路径变量
- 生产环境使用**绝对路径**或**环境变量**
- 确保上传目录有**写权限**

### 静态资源配置
```java
// WebConfig.java
registry.addResourceHandler("/uploads/**")
        .addResourceLocations("file:/absolute/path/to/uploads/");
```
- 路径格式：`file:/` + 绝对路径 + `/`
- 必须以 `/` 结尾

## 常见问题排查

### Q1: 上传后图片仍然无法显示
**检查**：
1. 后端日志中是否有"图片上传成功"和"实际保存路径"
2. 文件是否真的保存到磁盘：`ls -la <upload_path>/banners/`
3. 浏览器控制台网络请求的返回状态码

### Q2: 403 Forbidden
**检查**：
1. SecurityConfig 中 `/uploads/**` 是否在 `/api/**` 之前
2. 是否启用了 CORS（如果前端跨域）

### Q3: 404 Not Found
**检查**：
1. WebConfig 中配置的路径是否与实际文件路径一致
2. 上传目录是否存在且有文件

### Q4: 上传失败（前端）
**检查**：
1. 浏览器控制台网络请求的 URL 是否正确
2. Token 是否有效
3. 后端日志是否有错误信息
